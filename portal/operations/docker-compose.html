Guia de Uso dos Docker Compose Files

O DataFlow utiliza dois arquivos docker-compose para organizar a stack. Este documento explica como executar a infraestrutura e as aplicações, além de trazer comandos úteis para o dia a dia.

Arquivos Disponíveis

  • docker-compose.infrastructure.yml: infraestrutura (PostgreSQL, Redis, RabbitMQ, OpenTelemetry Collector, Prometheus, Grafana e exporters).
  • docker-compose.yml: aplicações (DataFlow API, Worker, Reporting Service e proxy Nginx).

Subindo os Serviços

Opção 1 — Infraestrutura e Aplicações em Etapas

1. Subir infraestrutura

docker compose -f docker-compose.infrastructure.yml --profile infra up -d

Serviços provisionados:

  • PostgreSQL — banco de dados (5432)
  • Redis — cache/coordenação (6379)
  • RabbitMQ — mensageria (5672 / 15672)
  • OpenTelemetry Collector (4317 / 55681)
  • Prometheus — métricas (9090)
  • Grafana + renderer — dashboards (3000)
  • Exporters — métricas de Redis (9121) e PostgreSQL (9187)
  • Rede dev_net para comunicação entre containers

2. Subir aplicações

docker compose --profile proxy --profile api --profile worker --profile reporting up -d
  • DataFlow API
  • DataFlow Worker
  • DataFlow Reporting Service
  • Nginx Proxy com certificados SSL

Opção 2 — Comandos sequenciais

# Infraestrutura
docker compose -f docker-compose.infrastructure.yml --profile infra up -d

# Aplicações
docker compose --profile proxy --profile api --profile worker --profile reporting up -d

Comandos Úteis

Inspeção e Logs

# Logs da infraestrutura
docker compose -f docker-compose.infrastructure.yml logs -f

# Logs das aplicações
docker compose logs -f

# Logs de um serviço específico
docker compose logs -f data-flow-api

Parada e Limpeza

# Parar aplicações
docker compose --profile proxy --profile api --profile worker --profile reporting down

# Parar infraestrutura
docker compose -f docker-compose.infrastructure.yml --profile infra down

# Parar tudo, removendo volumes
docker compose -f docker-compose.infrastructure.yml --profile infra down -v
docker compose --profile proxy --profile api --profile worker --profile reporting down -v

Rebuild das imagens

# Rebuild com up automático
docker compose --profile proxy --profile api --profile worker --profile reporting up -d --build

Ordem de Inicialização

Dica: Sempre suba a infraestrutura primeiro e aguarde os containers ficarem saudáveis antes de subir as aplicações.

  1. Infraestrutura (Banco, Cache, Mensageria, Observabilidade)
  2. Aplicações (API, Worker, Reporting, Proxy)

Endpoints Principais

  • Swagger API: https://api.local:8443/swagger
  • Swagger Reporting: https://reporting.local:8444/swagger
  • Grafana: http://localhost:3000
  • Prometheus: http://localhost:9090
  • RabbitMQ Management: http://localhost:15672 (admin / supersecret_admin)

Pós-subida

  • Verifique saúde dos containers: docker ps --format "table {{.Names}} {{.Status}} {{.Ports}}"
  • Confirme endpoints de health (/health) para API e Reporting
  • Acesse Grafana/Prometheus para validar métricas
