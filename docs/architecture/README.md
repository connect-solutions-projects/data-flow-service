# Documentação de Arquitetura

Esta pasta reúne artefatos técnicos que explicam as decisões de desenho do DataFlow.

## Conteúdo essencial

- `dataflow-technical-architecture.md` — visão técnica detalhada (monorepo, serviços, observabilidade).
- `decisoes-tecnicas.md` — decisões de arquitetura explicadas em linguagem acessível.

Estes documentos cobrem toda a arquitetura do sistema, incluindo:
- Organização do monorepo e benefícios
- Responsabilidades de cada serviço (API, Worker, Reporting)
- Estratégias para lidar com arquivos grandes
- Observabilidade (OpenTelemetry, Prometheus, Grafana)
- Escalabilidade e segurança
