Guia de Grafana e Prometheus

Como utilizar as ferramentas de observabilidade do DataFlow para monitorar métricas, criar dashboards e gerar relatórios.

1. Visão Geral

Prometheus

  • O que é: sistema de monitoramento e alerta baseado em séries temporais.
  • No DataFlow: armazena métricas coletadas do OpenTelemetry Collector.
  • Acesso: http://localhost:9090.

Grafana

  • O que é: plataforma de visualização e análise de dados.
  • No DataFlow: visualiza métricas do Prometheus, gera dashboards e imagens via renderer.
  • Acesso: http://localhost:3000 (credenciais padrão admin/admin).

2. Arquitetura de Observabilidade

API/Worker → OpenTelemetry → OTel Collector → Prometheus → Grafana
                                                     ↓
                                             Reporting Service
  1. Coleta — API e Worker publicam métricas/traces via OpenTelemetry.
  2. Agregação — OTel Collector recebe e processa os dados.
  3. Armazenamento — Prometheus faz scrape e persiste séries temporais.
  4. Visualização — Grafana consulta Prometheus e exibe dashboards.
  5. Relatórios — Reporting Service consolida métricas e imagens.

3. Configuração no Projeto

Prometheus

  • Hostname interno: docker-prometheus (rede dev_net).
  • Porta externa: 9090.
  • Datasource Grafana: http://docker-prometheus:9090.

Grafana

  • GF_RENDERING_SERVER_URL: http://docker-grafana-renderer:8081/render.
  • GF_RENDERING_CALLBACK_URL: http://docker-grafana:3000/.
  • Datasource Prometheus configurado no boot.

4. Métricas Disponíveis

ASP.NET Core

  • http_requests_total
  • http_request_duration_seconds
  • http_requests_in_progress

Custom (DataFlow)

  • dataflow_rate_limit_429_total
  • dataflow_deduplication_hits_total
  • Métricas de runtime do .NET (GC, threads, memória).

5. Consultas PromQL úteis

# Taxa de requisições (5 minutos)
rate(http_requests_total[5m])

# Latência p95
histogram_quantile(0.95, sum(rate(http_request_duration_seconds_bucket[5m])) by (le))

# Requisições por status
sum by(status) (rate(http_requests_total[5m]))

# Taxa de erros 5xx
sum(rate(http_requests_total{status=~"5.."}[5m]))

# Requisições ativas
sum(http_requests_in_progress)

# Rate limit 429
rate(dataflow_rate_limit_429_total[5m])

# Deduplicação
rate(dataflow_deduplication_hits_total[5m])

6. Criando dashboards no Grafana

  1. Acesse http://localhost:3000 e faça login.
  2. Crie um dashboard: Create → Dashboard.
  3. Adicione painéis escolhendo o datasource Prometheus.
  4. Configure consultas PromQL e tipo de visualização (time series, gauge, etc.).
  5. Salve o dashboard para reutilizar.

Exemplo: Painel de taxa de requisições com rate(http_requests_total[5m]).

7. Integração com Reporting Service

O Reporting Service consulta Prometheus, gera links do Grafana, exporta imagens (opcional) e produz relatórios markdown.

$body = @{
    job = "dataflow-api"
    window = "5m"
    outputDir = "docs"
} | ConvertTo-Json

Invoke-RestMethod -Method Post `
    -Uri "https://reporting.local:8444/reports/final" `
    -Body $body `
    -ContentType "application/json" `
    -SkipCertificateCheck

8. Troubleshooting

  • Sem métricas? Verifique o OTel Collector (docker logs otel-collector).
  • Renderer falhando? Reinicie grafana e grafana-renderer; confirme variáveis GF_RENDERING_*.
  • Permissões insuficientes? Altere a senha padrão do Grafana no primeiro login.
