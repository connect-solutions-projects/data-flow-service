Como adicionar OpenSSL ao PATH

Guia para habilitar o openssl.exe no PowerShell, evitando erros na geração de certificados.

Localização do OpenSSL

Instalações do Git for Windows geralmente incluem o executável em:

C:\Program Files\Git\usr\bin\openssl.exe

Método 1 — Temporário (sessão atual)

$env:Path += ";C:\Program Files\Git\usr\bin"

Verifique:

openssl version

Método 2 — Permanente

Via interface gráfica

  1. Abrir sysdm.cpl → aba Avançado.
  2. Clicar em Variáveis de Ambiente.
  3. Editar a variável Path (sistema) e adicionar C:\Program Files\Git\usr\bin.
  4. Aplicar e reiniciar o PowerShell.

Via PowerShell (Administrador)

[Environment]::SetEnvironmentVariable(
    "Path",
    [Environment]::GetEnvironmentVariable("Path", "Machine") + ";C:\\Program Files\\Git\\usr\\bin",
    "Machine"
)

Via PowerShell (Usuário atual)

[Environment]::SetEnvironmentVariable(
    "Path",
    [Environment]::GetEnvironmentVariable("Path", "User") + ";C:\\Program Files\\Git\\usr\\bin",
    "User"
)

Verificação

openssl version

Saída esperada:

OpenSSL 1.1.1w  11 Sep 2023

Alternativa

Utilize o Git Bash (já inclui o OpenSSL no PATH). O script generate-dev-certs.ps1 informa como proceder caso o binário não seja localizado.
